# Telegram-бот закрытого клуба по подписке

Асинхронный Telegram-бот для автоматизации платного доступа в закрытый Telegram-канал. Написан на Python с использованием **aiogram 3.x**, **PostgreSQL**, **SQLAlchemy** и полностью контейнеризирован с помощью **Docker**.

## 🚀 Функционал проекта

### 👤 Сценарий пользователя (User Flow)
- **Автоматическая регистрация:** При первом старте бот проверяет наличие пользователя в базе и сохраняет его данные (ID, username, имя, дата регистрации).
- **Главное меню:** Понятная навигация с помощью Inline/Reply кнопок («💰 Купить подписку», «ℹ️ О клубе», «👤 Мой профиль»).
- **Встроенная оплата:** Интеграция с официальными платежными провайдерами Telegram (ЮKassa / Тестовый провайдер) через нативные инвойсы.
- **Автоматический доступ:** Сразу после успешной оплаты бот меняет статус юзера на `active`, рассчитывает дату окончания подписки и генерирует уникальную одноразовую ссылку-приглашение в закрытый канал.
- **Личный кабинет:** Отображает текущий статус подписки и счетчик дней, оставшихся до её окончания.

### 👑 Сценарий администратора (Admin Flow)
- **Надежная защита:** Вход в панель администратора защищен кастомным `AdminMiddleware` (список ID админов кэшируется в память из БД). Для обычных пользователей команды админки полностью скрыты.
- **Панель статистики:** Мгновенный вывод аналитики в реальном времени: всего пользователей, новые за 24 часа, количество активных подписок и общая сумма заработанных средств.
- **Управление пользователями:** Возможность вручную забанить пользователя по ID или продлить ему подписку (+30 дней) прямо через чат бота.

### 🕒 Фоновые задачи (Background Tasks)
- **Мониторинг подписок:** Ежедневная фоновая задача (cron-job, запускается в 00:00), которая проверяет даты окончания подписок. Если срок истек, бот автоматически меняет статус в БД, исключает (кикает) пользователя из закрытого канала и отправляет ему уведомление в ЛС с предложением продлить доступ.

## 🛠️ Технологический стек и архитектура

- **Язык программирования:** Python 3.11+
- **Фреймворк:** `aiogram` (v3.x)
- **База данных:** `PostgreSQL` (Реляционная СУБД)
- **ORM:** `SQLAlchemy` (Асинхронный движок) + паттерн Data Access Objects (DAO)
- **Планировщик задач:** `asyncio` / `APScheduler`
- **Контейнеризация:** `Docker` & `Docker Compose`

## 📦 Установка и запуск проекта

### 1. Клонирование репозитория
```bash
git clone https://github.com
cd tg13
```

### 2. Настройка переменных окружения
Создайте файл `.env` в корневой директории проекта на основе шаблона `.env.example`:
```env
BOT_TOKEN=123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ
PAYMENTS_TOKEN=1234567:TEST:12345
ADMINS=12345678,98765432

DB_USER=postgres
DB_PASSWORD=my_secure_password
DB_NAME=club_bot
DB_HOST=db
DB_PORT=5432
```

### 3. Запуск через Docker Compose
Запустите базу данных и контейнер с ботом одной командой (данные БД будут сохраняться на диске хоста):
```bash
docker-compose up --build -d
```

## 🔒 Безопасность и лучшие практики
- **Разделение конфигурации:** Все конфиденциальные данные (токены, пароли к БД) полностью изолированы от исходного кода и управляются строго через переменные окружения (`.env`).
- **Сохранение данных (Data Persistence):** Данные СУБД PostgreSQL соединены с диском основной машины через Docker `volumes`. Это гарантирует, что при перезапуске контейнеров или обновлении кода база данных не сотрется.
